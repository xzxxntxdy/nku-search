# 南开 Web 搜索引擎演示视频文案与操作指示

> 建议视频时长：12 到 15 分钟。  
> 录屏方式：全屏录制浏览器 + 命令行窗口。  
> 讲解方式：每一段先按“操作指示”完成屏幕动作，再照着“旁白”念。  
> 当前不演示打包提交，重点展示浏览器界面中的数据规模、搜索服务、个性化和推荐功能。

## 0. 录制前准备

### 0.1 打开两个窗口

**操作指示**

1. 打开一个 Windows CMD 窗口，命名为“服务窗口”。
2. 打开浏览器，准备访问 `http://127.0.0.1:8000`。
3. 如果已经有服务在运行，可以先直接进入 0.3 检查；如果不确定，按下面命令重新启动。

**服务窗口执行**

```bat
cd /d D:\hw4
docker compose up -d elasticsearch
.\.venv\Scripts\python.exe -B -m nku_search.serve --host 127.0.0.1 --port 8000
```

**注意**

- 这条 `serve` 命令会占住当前窗口，不要关闭。
- 如果提示端口 8000 已被占用，说明服务可能已经启动；可以直接浏览器访问 `http://127.0.0.1:8000/health` 检查。

### 0.2 如果需要重新构建前端

**操作指示**

如果你刚修改过前端，单独打开一个 CMD 窗口执行：

```bat
cd /d D:\hw4\frontend
npm.cmd run build
cd /d D:\hw4
```

**旁白**

这里我使用 React、Vite 和 Ant Design 构建前端页面。构建后的静态文件由 FastAPI 直接托管，所以系统运行时前端和后端都在同一个端口 `8000` 上，不需要两个服务端口。

### 0.3 快速打开首页

**操作指示**

打开浏览器访问：

```text
http://127.0.0.1:8000
```

**预期结果**

- 进入南开资源搜索界面。
- 搜索框默认为空。
- 页面自动加载全库结果，顶部显示总结果数。
- 左侧筛选区显示文件类型、站点和主题数量。

**旁白**

正式演示前先打开系统首页。当前前端由 FastAPI 直接托管，前后端在同一个 `8000` 端口。搜索框默认留空，页面会加载全库结果和分面统计，可以直接从界面看到数据规模和各类资源数量。

## 1. 开场介绍，约 1 分钟

**操作指示**

1. 浏览器打开：`http://127.0.0.1:8000`
2. 停留在首页，展示左侧菜单：搜索、主题、文档、历史、账号。

**旁白**

大家好，我演示的是信息检索第四次作业：南开 Web 搜索引擎。这个项目定位为综合性的南开资源搜索平台。系统覆盖南开主站、新闻站、院系站、教学招生、科研平台、图书资源、校园服务，以及南开动漫资源站。

技术栈采用 Scrapy、Elasticsearch、FastAPI、SQLite、React、Vite 和 Ant Design。整体流程是：Scrapy 负责大规模抓取网页和文档，索引阶段解析 HTML、PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX 等内容并计算 PageRank，然后写入 Elasticsearch；FastAPI 提供搜索 API、账号、日志、快照和推荐接口；前端用 React 和 Ant Design 做统一的搜索界面。

左侧这里是系统的主要功能入口，包括综合搜索、主题分类、文档检索、历史记录和账号登录。

## 4. 爬虫设计与抓取计划，约 1.5 分钟

**操作指示**

1. 在主界面点击左侧“主题”。
2. 展示八个主题板块卡片。
3. 指出每个卡片上的“已索引”数量。
4. 点击“动漫资源”或其他主题的“搜索该主题”按钮。

**旁白**

这里是爬虫计划和主题入口。作业要求至少 10 万页，系统默认计划目标是 16 万页，并且按主题板块分配预算。八个板块分别是新闻资讯、学校门户、院系学科、教学招生、科研学术、图书资源、动漫资源和校园服务。每个卡片上的“已索引”数量直接展示当前索引中该主题的实际数据规模。

爬虫使用 Scrapy 的 CrawlSpider、Rule、LinkExtractor、Pipeline 和 Middleware。它支持 robots 协议、AutoThrottle、HTTP cache、JOBDIR 断点续爬、按板块预算控制、低价值列表页过滤、超大文档下载拦截，以及已有 JSONL 的续爬去重。

特别是动漫板块，系统专门覆盖了 `12club.nankai.edu.cn`，并针对 Next.js 页面补充了资源分页和详情链接提取逻辑，所以不只是抓普通静态 HTML。

现在回到前端的“主题”页，可以看到每个主题板块都可以直接进入搜索。点击某个主题时，系统会自动带上中文 `category` 主题过滤条件；内部的 `section` 仍保留给爬虫预算和索引分区使用，但前端不再重复展示。

## 5. 综合搜索与分页，约 1.5 分钟

**操作指示**

1. 点击左侧“搜索”。
2. 搜索框输入：`南开`
3. 点击“搜索”。
4. 展示结果总数、标题、URL、摘要、文件类型标签、主题标签、快照按钮。
5. 点击分页器的最后一页，或者在跳页框输入最后页页码。
6. 展示最后一页能正常返回结果，不报 500。

**旁白**

现在演示综合搜索。输入“南开”后，系统会通过 FastAPI 调用 Elasticsearch 查询。结果中展示标题、URL、摘要、文件类型、主题分类和网页快照入口。

排序方面，Elasticsearch 查询融合了文本相关性和 PageRank。对于本地兜底后端，系统实现了 BM25F、短语匹配、标题命中、URL 命中、PageRank、新鲜度和个性化 boost。ES 后端则用 `multi_match` 检索标题、正文、锚文本和 URL，并用 `field_value_factor` 将 PageRank 加入排序。

这里我再跳到最后一页。之前深分页和超长正文高亮可能导致 500，现在已经修复：ES 查询设置了 `max_result_window=200000`，同时限制高亮分析长度，并且 API 响应不取回超大的正文、锚文本和出链字段。因此最后一页也能正常返回。

## 6. 分面筛选，约 1 分钟

**操作指示**

1. 保持“南开”搜索结果页。
2. 看左侧“筛选”区域。
3. 点击一个文件类型，例如 `html` 或 `pdf`。
4. 再点击一个主题筛选。
5. 点击“重置筛选”，恢复完整搜索结果。

**旁白**

这里是分面筛选。系统会对当前搜索结果做聚合统计，支持按文件类型、站点和主题分类过滤。左侧数量表示当前查询命中的结果内有多少对应类型，并非全库固定数量；例如某个查询下 `pptx` 可能只显示 5 个，清空查询后全库会显示完整的 PPT 和 PPTX 数量。真实 ES 后端中，这部分是通过 Elasticsearch aggregations 实现的；本地后端则通过 `LocalInvertedIndex.facets()` 实现。

点击筛选项以后，前端会把 `filetype`、`site` 或 `category` 参数重新提交到 `/api/search`。重置筛选会清空文件类型、站点和主题条件，回到当前关键词的完整结果。这和现代搜索引擎的左侧筛选栏逻辑一致。

## 7. 六类高级查询，约 3 分钟

### 7.1 站内查询

**操作指示**

1. 回到“搜索”。
2. 搜索框输入：`site:news.nankai.edu.cn 南开`
3. 点击搜索。
4. 展示结果 URL 都来自新闻站。

**旁白**

第一类是站内查询。这里使用高级查询语法 `site:news.nankai.edu.cn 南开`，把站点限制为南开新闻站。系统会在查询解析阶段识别 `site:`，自动兼容带协议和不带协议的域名写法，并在 Elasticsearch 查询中加入 URL 前缀过滤。搜索结果左侧也提供站点分面筛选，可以从结果聚合中继续限定来源站点。

### 7.2 文档查询

**操作指示**

1. 点击左侧“文档”。
2. 默认展示 `pdf`。
3. 切换 `docx`、`xlsx`，再切回 `pdf`。
4. 展示文档标题、URL、摘要、快照按钮和分页。

**旁白**

第二类是文档查询。系统支持 PDF、DOC、DOCX、XLS、XLSX 等附件。爬虫下载文档后，索引阶段会用 `pdfplumber`、`python-docx` 和 `openpyxl` 提取文本。当前数据中 PDF 有 4846 条，DOCX 有 1436 条，XLSX 有 763 条。

文档页本质上也是 `/api/search`，只是默认带上 `filetype` 过滤，并且支持分页。

### 7.3 短语查询

**操作指示**

1. 点击“搜索”。
2. 搜索框输入：`"南开大学"`
3. 保持模式为“自动”，点击搜索。
4. 再去掉引号并拆成两个词，搜索：`南开 大学`
5. 对比两次结果数量。

**旁白**

第三类是短语查询。短语查询强调多个词按固定顺序连续出现。这里 `"南开大学"` 会作为一个完整短语匹配；去掉引号并拆成 `南开 大学` 后，会按普通关键词查询，结果范围会扩大。系统在查询解析中识别英文双引号和中文引号，也可以通过 `mode=phrase` 指定短语模式。ES 后端使用 `multi_match` 的 phrase 查询，本地后端会检查完整短语是否出现在标题或正文中。

### 7.4 通配查询

**操作指示**

1. 搜索框输入：`南开*`
2. 模式选择“通配”。
3. 点击搜索。
4. 再试一个：`计?`

**旁白**

第四类是通配查询。星号代表多个字符，问号代表一个字符。比如 `南开*` 可以匹配以南开开头的 token，`计?` 可以匹配类似“计网”“计算”这样的词形。ES 后端使用 wildcard 查询，本地后端会把通配符转换成正则表达式匹配词表。

### 7.5 正则查询

**操作指示**

1. 搜索框输入：`/南.*/`
2. 模式选择“正则”。
3. 点击搜索。
4. 展示有结果返回。

**旁白**

第五类是正则查询。这里 `/南.*/` 表示匹配以“南”开头的 token。系统支持 `/pattern/` 形式，也支持手动选择正则模式。ES 后端使用 regexp 查询，本地后端使用 Python 的正则匹配。

### 7.6 查询日志

**操作指示**

1. 点击左侧“历史”。
2. 展示刚才搜索过的查询词、模式、站点、文件类型、结果数量和时间。
3. 点击“刷新”。

**旁白**

第六类是查询日志，也就是历史记录。每次搜索都会写入 SQLite 的 `query_logs` 表，包括用户、查询词、模式、站点、文件类型、结果数量和时间。前端历史页通过 `/api/history` 展示这些记录。

### 7.7 网页快照

**操作指示**

1. 回到搜索结果页。
2. 找一条结果，点击“快照”。
3. 新标签页打开快照内容。

**旁白**

网页快照是搜索引擎在收录网页时保存的一份副本。系统在索引阶段把 HTML 或文档文本保存到 `data/snapshots`，通过 `/snapshot/{doc_id}` 访问。这样即使原始网页变化或暂时打不开，用户仍然可以查看当时抓取到的内容。

## 8. 个性化查询，约 1.5 分钟

**操作指示**

1. 点击左侧“账号”。
2. 选择“注册账号”。
3. 输入一个临时用户名，例如：`demo_user_001`
4. 密码输入：`pass1234`
5. 确认密码再次输入：`pass1234`
6. 兴趣词输入：`人工智能 信息检索 动漫`
7. 点击“注册并登录”。
8. 注册成功后展示左侧“账号状态”和右侧“个性化设置”。
9. 在“个性化设置”中把兴趣词改成：`人工智能 信息检索 图书馆`
10. 点击“保存兴趣词”。
11. 回到“搜索”，搜索：`南开`
12. 再搜索：`人工智能`
13. 点击一个搜索结果。
14. 点击“历史”，展示该用户的查询记录。

**旁白**

现在演示个性化查询。注册用户时需要填写用户名、密码、确认密码，也可以填写兴趣词，比如人工智能、信息检索和动漫。注册成功后系统会自动登录，并在账号页展示当前登录用户和兴趣词。兴趣词可以继续编辑，更新后会保存到 SQLite 的 `users` 表。

登录后，用户每次搜索都会写入查询历史，点击结果会写入点击日志。排序时，系统会把注册兴趣词和历史查询一起作为用户兴趣输入。ES 后端会把这些词作为 should boost 加入查询，本地后端会通过 `infer_interest_score()` 计算个性化得分。

这说明不同登录用户在相同查询下，可以根据自己的兴趣和历史得到不同的排序倾向。

## 9. 搜索联想推荐，约 1 分钟

**操作指示**

1. 回到“搜索”。
2. 先展示搜索框下方的“推荐”词条。
3. 点击一个推荐词，展示系统自动搜索。
4. 再在搜索框输入一个字，例如：`南`
5. 不要立刻按搜索，等待下拉联想出现。
6. 点击一个联想词。
7. 展示系统自动搜索。

**旁白**

这里演示个性化推荐中的搜索联想。搜索框下方会直接展示推荐词，输入前缀后也会出现下拉联想。联想词来源有三类：第一是索引阶段从标题和正文中抽取的高频词，第二是匿名全局历史查询，第三是当前登录用户的个人历史。

对应后端接口是 `/api/suggest`。全局联想词保存在 SQLite 的 `suggestions` 表，个人联想词来自当前用户自己的 `query_logs`。用户登录后查询越多，搜索界面中越容易看到和自己历史相关的推荐词；游客不会看到登录用户的个人查询词。

## 附录：常见问题处理

### 1. `/health` 或 `/api/health` 显示 LocalSearchBackend

**原因**：Elasticsearch 没启动、9200 不可访问，或者 8000 上还残留着之前启动的旧 Python 进程。  
**处理**：

```bat
cd /d D:\hw4
docker compose up -d elasticsearch
```

如果 8000 被旧进程占用，先结束旧进程：

```bat
for /f "tokens=5" %a in ('netstat -ano ^| findstr :8000 ^| findstr LISTENING') do taskkill /PID %a /F
```

等待 10 秒后重启 FastAPI。启动器会检查 `frontend/dist` 是否过期，必要时自动执行前端构建：

```bat
.\.venv\Scripts\python.exe -B -m nku_search.serve --host 127.0.0.1 --port 8000
```

再访问 `http://127.0.0.1:8000/health`，确认 `backend` 是 `SearchBackend`，`documents` 是 `135779`。

### 2. 浏览器还是旧页面

**处理**：按 `Ctrl + F5` 强制刷新。当前后端返回前端入口时已经加了 `Cache-Control: no-store`，正常刷新后会拿到最新构建产物。

### 3. npm 在 PowerShell 报执行策略错误

**处理**：使用 CMD，并运行：

```bat
npm.cmd run build
```

### 4. 不想现场跑完整测试

**处理**：跳过命令行测试，直接展示浏览器里的搜索结果总数、分面数量、文档页、历史页、账号页和推荐词。
