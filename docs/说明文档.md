# 南开 Web 搜索引擎说明文档

## 1. 项目目标

本项目实现一个综合南开校内资源搜索引擎，面向南开主站、新闻站、院系站、教学招生、科研学术、图书馆、动漫资源和校园服务等来源提供检索服务。系统覆盖课程要求中的网页抓取、文本索引、查询服务、个性化查询、Web 界面和个性化推荐。

技术栈：

```text
Scrapy + Elasticsearch + FastAPI + SQLite + React + Vite + Ant Design
```

项目参考 Scrapy、Whoosh、FastAPI Full Stack Template、Ant Design Pro、Elastic Search UI、Haystack、Meilisearch 的设计思想，在本项目中重写为统一的课程工程。`references/` 仅作为本地参考仓库，不提交到 Git。

## 2. 系统架构

```text
Scrapy crawler
  -> JSONL crawled pages
  -> clean / dedupe / document parsing
  -> snapshot + PageRank + suggestion terms
  -> Elasticsearch index
  -> FastAPI JSON API + SQLite user/log storage
  -> React + Ant Design single-port search console
```

核心数据流：

1. `nku_search.crawl` 使用 Scrapy 抓取网页和附件，输出 JSONL。
2. `nku_search.clean` 过滤服务页、空壳页、异常状态和重复 URL。
3. `nku_search.index` 读取 JSONL，计算 PageRank，保存快照，批量写入 Elasticsearch。
4. `nku_search.web` 提供搜索、联想、账号、日志、点击和快照 API。
5. `frontend/src/SearchConsole.tsx` 提供搜索、主题、文档、历史和账号界面。

## 5. 查询与排序

Elasticsearch 查询由 `nku_search/query.py` 构建，主要字段包括：

- `title^3`
- `anchors^2`
- `text`
- `url`
- `pagerank`
- `filetype`
- `section`
- `category`

排序融合：

```text
Elasticsearch text score + PageRank field_value_factor + user interest boost
```

本地兜底后端 `LocalSearchBackend` 实现 BM25F、短语匹配、标题命中、URL 命中、PageRank、新鲜度和个性化 boost，用于测试和 ES 不可用时的小规模演示。

查询示例：

| 类型 | 示例 |
|---|---|
| 普通查询 | `南开 大学` |
| 短语查询 | `"南开大学"` |
| 中文引号短语 | `“南开大学”` |
| 站内查询 | `site:news.nankai.edu.cn 南开` |
| 文档查询 | `filetype:pdf CARSI` |
| 标题查询 | `title:人工智能 南开` |
| 通配查询 | `南开*`、`计?` |
| 正则查询 | `/南.*/` |

普通查询将输入视为关键词组合；短语查询要求多个词按照固定顺序连续出现。

## 6. Web 与 API

前端由 FastAPI 托管，运行时只有一个端口：

```text
http://127.0.0.1:8000
```

主要 API：

- `GET /api/search`
- `GET /api/suggest`
- `GET /api/topics`
- `GET /api/stats`
- `GET /api/crawl-plan`
- `GET /api/history`
- `GET /api/pipeline`
- `GET /api/references`
- `POST /api/register`
- `POST /api/login`
- `POST /api/profile`
- `POST /api/click`
- `GET /snapshot/{doc_id}`
- `GET /health` / `GET /api/health`

前端页面：

- 搜索：关键词、模式、推荐词、结果列表、分页、分面筛选和快照入口。
- 主题：按新闻、主站、院系、教学、科研、图书、动漫、服务展示索引数量。
- 文档：按 PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX 等类型筛选。
- 历史：展示查询日志。
- 账号：注册、登录、兴趣词更新和退出登录。

## 7. 运行方法

以下命令均假定在 Git 仓库根目录执行。首次使用时先 clone 项目并安装依赖：

```powershell
git clone https://github.com/xzxxntxdy/nku-search.git
cd nku-search
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
cd frontend
npm install --cache ..\.npm-cache
cd ..
```

Linux/macOS 使用 `source .venv/bin/activate` 激活虚拟环境。虚拟环境激活后，后续命令统一使用 `python -m ...`。

启动服务：

```powershell
docker compose up -d elasticsearch
python -B -m nku_search.serve --host 127.0.0.1 --port 8000
```

健康检查：

```powershell
curl http://127.0.0.1:8000/health
```

`backend` 为 `SearchBackend` 时表示已连接 Elasticsearch；`backend` 为 `LocalSearchBackend` 时表示使用内置样例数据。

建立索引：

```powershell
python -B -m nku_search.index --reset
```

如果本机有真实爬取数据：

```powershell
python -B -m nku_search.index --input data/crawl/pages_160k_clean.jsonl --reset
```
