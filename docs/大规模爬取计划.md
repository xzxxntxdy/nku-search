# 南开资源大规模爬取计划

## 目标

默认规模设置为 160000 页，高于作业最低要求 100000 页。开发和演示阶段可使用样例数据或按板块小规模抓取；正式校园网抓取时使用同一套板块预算、断点续爬和 Elasticsearch 批量索引流程。

## 板块预算

| key | 主题 | 目标页数 | 种子方向 | 策略 |
|---|---|---:|---|---|
| news | 新闻资讯 | 36000 | 南开新闻网、学校要闻、综合新闻、媒体南开 | 新闻页深度 8，低并发，保留历史新闻 |
| main | 学校门户 | 14000 | 主站、学校概况、机构、人才培养 | 主站作为权威入口，控制重复列表页 |
| schools | 院系学科 | 42000 | 计算机、数学、物理、化学、环境、法学、经济、商学院、历史等学院站 | 多域名分散抓取，支撑教师、科研、培养方向查询 |
| academic | 教学招生 | 26000 | 教务、研究生院、研招、本科招生 | 文档优先，重点抓 PDF/DOCX/XLSX 附件 |
| research | 科研学术 | 18000 | 科研处、社科处、学术平台 | 学术通知、项目、成果、实验室信息 |
| library | 图书资源 | 8000 | 图书馆、数据库、电子资源 | 文档和服务指南优先 |
| anime | 动漫资源 | 12000 | 南开动漫资源站 `12club.nankai.edu.cn` | 动漫资源站、社团活动、资源索引和兴趣内容 |
| services | 校园服务 | 4000 | 信息化服务 | 校园网络、账号和生活服务补充 |

## 执行命令

查看全部板块：

```powershell
python -m nku_search.crawl --list-sections
```

完整计划：

```powershell
python -m nku_search.crawl --max-pages 160000 --output data/crawl/pages.jsonl
python -m nku_search.index --input data/crawl/pages.jsonl --reset
```

按板块分批，便于恢复和审查：

```powershell
python -m nku_search.crawl --section news --output data/crawl/news.jsonl
python -m nku_search.crawl --section schools --output data/crawl/schools.jsonl
python -m nku_search.crawl --section anime --output data/crawl/anime.jsonl
python -m nku_search.crawl --section academic --section research --output data/crawl/academic_research.jsonl
```

扩容到约 300000 页：

```powershell
python -m nku_search.crawl --scale-factor 2 --output data/crawl/pages_300k.jsonl
```

## Scaling 设计

- 抓取层：Scrapy `JOBDIR` 支持断点续爬，`HTTPCACHE` 降低重复请求，`AutoThrottle` 根据站点响应自动调速。
- 板块层：`section` 和 `category` 写入每条 JSONL，索引后可直接做主题过滤、Facet 聚合和 UI 分类浏览。
- 索引层：Elasticsearch mapping 已加入 `section/category/filetype/url` keyword 字段，支持按主题过滤与聚合。
- 排序层：本地 BM25F 与 Elasticsearch 查询均保留 `section/category` 过滤；排序仍融合 BM25F、短语匹配、标题命中、URL 命中、PageRank、新鲜度和个性化 boost。
- 运行层：可按板块生成多个 JSONL，合并后统一索引；也可以在正式环境中分批追加到同一索引。

## 礼貌抓取

默认遵守 `robots.txt`，每域并发 1-2，下载延迟 0.7-1.0 秒，避免对学校站点造成压力。正式抓取应优先在校园网低峰时段执行。
